## Methodology (Leak-Free)

**Critical: This benchmark uses a SPLIT-FIRST design to prevent train/test leakage.**

1. **Dataset loaded** as pandas DataFrame
2. **Target detection**: manual — column `Class`
3. **Train/test split**: 80/20 stratified split (seed 42) BEFORE any pruning
4. **Column reordering**: target moved to last column (standard DataPruning convention)
5. **Preprocessing**: standardized preprocessing and feature alignment applied independently to train and test sets
6. **Feature alignment**: train/test feature spaces are intersected to ensure consistent dimensionality for model evaluation
7. **Dataset optimization**: the DataPruning production optimization pipeline processes the **TRAINING SET ONLY** — the test set is NEVER touched by optimization logic
8. **Safety layer**: reduction cap and class preservation applied to training set only
9. **Model training**: models are trained on the original training set AND the optimized training set
10. **Evaluation**: BOTH models are evaluated on the SAME untouched test set — the only variable is the training data

### Why This Matters

If the optimization used the full dataset (including test rows), the test evaluation would be invalid —
the model would appear to perform better simply because the test set was cherry-picked.
By splitting FIRST and optimizing ONLY the training set, we ensure:
- The test set represents genuinely unseen data
- Any improvement (or preservation) of metrics is due to better training data quality
- Results are scientifically defensible for ML peer review
